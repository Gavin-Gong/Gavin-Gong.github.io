后端开发学习之旅

2016-12-04
FE
Node

前言

大概从10月30开始学习Node后端, 到现在差不多一个月左右(中间玩了几天), 在某些方面我从来不是一个循规蹈矩的人, 所以我一开始就在按照自己的想法写, 写的同时也参考了很多教程, 源码, google某些功能是怎么实现的. 毕竟很多东西都是大同小异, 一法通万法通. 当然这也踩了不少的坑, 导致好几次都大面积重构代码.

梳理下遇到的知识点和坑

MiddleWare

按照我的理解, 所谓的中间件就好像是一个个不同特质的过滤器, 当然有时候也可以充当开关的作用, 可以对传过来的参数进行处理(赋值, 添加方法……)之后然后再传递给下一个中间件, 有时候也可能立即终止了. 看下express-flash的代码.

session

session 常常拿来与 cookie 进行比较 cookie数据是存在客户端的, 而session数据则是存在服务的,
这样说还是不够清楚, session的主要数据虽然是存在服务器, 但是也会给网页设置一个sessionid (相当于一把钥匙), 当用户请求的时候就会携带sessionId, 当服务器读取到sessionId就是知道是谁, 不会把别人的个人数据发给你了.

Mongdb 数据库

说实话, 这是我第一次接触数据库,感觉数据库也是很深的坑, 在此不做展开.

Router & Controller

这两者之间一度让我懵逼, link controller -> 基于请求的行为, 某种意义上的中间件 Rouetr -> 指定 url 和 对应的controller

表单验证

在做表单验证一个, 遭遇一个深坑, 刚开始我把表单验证写在controllers里面, 但是写着写着我发现, mongoose里面自带验证, -> 我当时就想表单验证不会是要组织到Model里面吧? -> 然后我又考虑了下Model是最贴近存储数据的 -> Google到一些写php的回答, 建议写到Model里面, 我想了想尽管语言不同, 但是都是MVC, 应该是同样适用的. 后来请教一位前辈 :Model是禁止的做法, 写到middleware 才是推荐做法, 然后我又重构一次代码 .

git 报错

error: RPC failed; result=52, HTTP code = 0fatal: The remote end hung up unexpectedly 解决方法

git config --global http.postBuffer 524288000

后话

好记性不如烂笔头, 当时写的时候想着, 等项目写完之后一起做下笔记总结, 但是到现在已经忘得差不多了.
